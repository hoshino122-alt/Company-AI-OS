# Company AI OS Production System
# Production Bible

Version : 0.1

Last Update : DAY29

---

# Purpose

Company AI OSを100日間、一貫した品質で制作するための制作ルールを管理する。

Production Systemは、
制作効率だけではなく、

・世界観の統一
・品質の維持
・制作速度の向上
・再利用可能なテンプレート作成

を目的とする。

---

# Development Policy

・制作しながらProduction Systemを育てる

・気付いたことはProduction Bibleへ追加する

・テンプレート化できるものはテンプレート化する

・Company AI OSの世界観を統一する

・Production System自身もCompany AI OSと共に成長する

---

# Rule000

Production Systemは完成させるものではない。

制作しながら改善し、
気付いたことをProduction Bibleへ反映する。

Production System自身も開発対象である。

---

# Rule001

Ren'Py画像フォルダはDAY単位で管理する。

images/dayXX/dayXX_01.png

例

images/day29/day29_01.png

---

# Rule002

音声フォルダもDAY単位で管理する。

voice/dayXX/dayXX_01.wav

例

voice/day29/day29_01.wav

---

# Rule003

Ren'Py画像名は

dayXX_01

dayXX_02

・・・

dayXX_07

で統一する。

---

# Rule004

音声ファイル名も画像と同じ番号で統一する。

dayXX_01.wav

dayXX_02.wav

・・・

dayXX_07.wav

---

# Rule005

毎日の制作順は固定する。

① Ren'Py

② ナレーション原稿

③ スライド画像

④ YouTube

⑤ note

⑥ GitHub

制作順は変更しない。

---

# Rule006

YouTube・note・GitHubは
Ren'Py完成後に制作する。

ストーリーを先に完成させてから
派生コンテンツを作成する。

---

# Rule007

AI社員のデザインは統一する。

Company AI OSでは

AURA CORE

をAI社員の共通デザインとする。

社員ごとの差は

・ホログラムUI

・役割表示

・周辺インターフェース

で表現する。

---

# Rule008

画像プロンプトは

Base Prompt

Role Prompt

Slide Prompt

の3階層構造で作成する。

Base Promptは100日間共通とする。

---

# Rule009

YouTube概要欄は以下の構成で統一する。

・概要

・今回の内容

・Company AI OS紹介

・リンク

・開発環境

・ハッシュタグ

---

# Rule010

GitHub開発日誌は

Overview

What I Built

Learning

Next Step

の4構成で統一する。

---

# Rule011

note記事は

・導入

・今回作ったもの

・開発中に考えたこと

・次回予告

・まとめ

の流れで作成する。

---

# Rule012

字幕は短く、
スマートフォンでも読みやすい長さにする。

文章は一文を短く区切る。

---

# Rule013

スライド切替時は
不要なpauseを入れない。

映像が滑らかになる構成を優先する。

---

# Rule014

BGMとナレーションの音量バランスは

BGM

volume 0.4

音声

volume 4.0

を標準とする。

---

# Rule015

GitHubでは
README.mdはプロジェクト全体のみ管理する。

DAYごとのREADME.mdは作成しない。

docs/diary/DAYXXX.md

を正式な開発日誌とする。

---

# Rule016

Production Bibleは
「今分かっている最善」を記録する。

ルールは固定ではなく、
より良い方法が見つかれば更新する。

---

# Rule017

テンプレートを先に作らない。

実際に数日運用し、
繰り返し使われたものだけをテンプレート化する。
